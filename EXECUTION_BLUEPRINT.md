### 2.4 Agent 04 — Trainee-Production Matching Agent

**Frequency**: Continuous (embedded in Cineterns portal)
**Owner**: Placement Manager + Tech Lead
**Output**: Real-time candidate suggestions for production briefs

**Data Inputs**
- Trainee profile database (Cineterns): skills, availability, previous placements, ratings
- Production company briefs: role requirements, timeline, budget, location
- Tier tracking: tier 1 (initial placement), tier 2 (repeat placements), tier 3 (leadership roles)

**Matching Algorithm**
1. **Skills matching**: Exact match on department + role + experience level
2. **Availability filtering**: Remove trainees on active productions
3. **Tier optimization**: Suggest tier 2/3 trainees first (higher retention, faster placement)
4. **Rating scoring**: Weight by production company ratings, trainee reliability scores
5. **Diversity signaling**: Bias toward underrepresented demographics if multiple equal candidates
6. **Ranking**: Return top 5 candidates with match scores and rationales

**Workflow Steps**
1. **Brief submission** (human): Production company enters role requirement in portal
2. **Instant matching** (automated): Agent generates candidate list within 2 minutes
3. **Company review** (human): Production company reviews profiles, makes selection
4. **Selection** (human): Production company clicks "confirm placement"
5. **Tier advancement** (automated): Trainee tier automatically advances if placement confirmed
6. **Tracking** (automated): Placement logged, dates tracked, feedback request queued

**Success Metrics**
- Time to candidate list: Target = 2 minutes (vs. 24-48 hours manual email search)
- Acceptance rate: Target = 75% (candidate agrees to placement)
- Placement cycle time: Target = 3-5 days (vs. 10-14 days manual)
- Tier advancement: Target = 60% of trainees reach tier 2 within 18 months

**Platform**: Supabase + Claude API + Next.js Cineterns interface
